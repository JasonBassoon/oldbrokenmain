Detailed Implementation Documentation

Step 1

Install KVM/QEMU Virtualization Stack

Commands:
sudo apt update && sudo apt upgrade -y sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils virt-manager virtinst cpu-checker kvm-ok sudo usermod -aG libvirt,kvm $USER sudo systemctl enable --now libvirtd lsmod | grep kvm
Rationale:
KVM provides hardware-accelerated virtualization on Linux, essential for running multiple VMs efficiently. This foundation enables the entire SOC lab infrastructure.
Evidence & Verification:
Verified VT-x virtualization support enabled. KVM modules loaded successfully (kvm_intel). System ready for VM deployment.
Step 2

Create Virtual Networks

Commands:
sudo virsh net-list --all sudo virsh net-start default sudo virsh net-autostart default sudo virsh net-define socnet.xml sudo virsh net-start socnet sudo virsh net-autostart socnet
Rationale:
Proper network segmentation isolates the SOC lab from the host system while allowing controlled internet access through pfSense. The socnet provides an isolated environment for attack simulations.
Evidence & Verification:
Created socnet (10.10.10.0/24) with DHCP reservations for all lab VMs. Default NAT network active for WAN connectivity.
Step 3

Download Required ISOs

Commands:
mkdir -p ~/ISOs cd ~/ISOs wget https://atxfiles.netgate.com/mirror/downloads/pfSense-CE-2.7.2-RELEASE-amd64.iso.gz gunzip pfSense-CE-2.7.2-RELEASE-amd64.iso.gz wget https://releases.ubuntu.com/22.04/ubuntu-22.04.3-live-server-amd64.iso wget https://cdimage.kali.org/kali-2025.1/kali-linux-2025.1-installer-amd64.iso
Rationale:
Gathering installation media for all VM operating systems. Using evaluation versions for Windows systems to stay within licensing compliance.
Evidence & Verification:
Downloaded pfSense CE 2.7.2, Ubuntu Server 22.04 LTS, and Kali Linux 2025.1. Windows ISOs to be downloaded from Microsoft Evaluation Center.
Step 4

Deploy pfSense Firewall VM

Commands:
sudo virt-install --name pfsense --ram 1024 --vcpus 1 --cpu host-passthrough --disk path=/var/lib/libvirt/images/pfsense.qcow2,size=10,format=qcow2,bus=virtio --network network=default,model=virtio --network network=socnet,model=virtio --graphics vnc,listen=127.0.0.1 --cdrom ~/ISOs/pfSense-CE-2.7.2-RELEASE-amd64.iso --os-variant freebsd13.0 --boot uefi
Rationale:
pfSense serves as the network gateway, providing firewall capabilities, IDS/IPS through Suricata, and logging for network traffic analysis. This is the first line of defense in the SOC lab.
Evidence & Verification:
pfSense VM created with dual NICs (WAN on NAT, LAN on socnet). Configured firewall rules and enabled Suricata IDS with ET Open ruleset.
Step 5

Validate Network Connectivity

Commands:
sudo virsh list ping -c 4 10.10.10.1 sudo virsh net-dhcp-leases socnet
Rationale:
Testing ensures proper network segmentation and internet access before deploying additional VMs. Validates pfSense routing and firewall configuration.
Evidence & Verification:
pfSense responsive on 10.10.10.1. WAN interface receiving DHCP from NAT network. LAN interface serving DHCP to socnet. Internet connectivity validated through pfSense gateway.
